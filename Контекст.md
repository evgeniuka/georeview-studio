# Контекст проекта: GeoReview Studio v083

Этот файл можно целиком вставить в новый Codex-чат, чтобы быстро восстановить суть проекта.

## Что это

GeoReview Studio v083 — local-first GIS review workbench для infrastructure risk indicators, data-quality evidence и portfolio artifacts. Проект помогает анализировать pedestrian access review candidates, особенно для Safe Access Israel / Kfar Saba style workflows.

Это не crash prediction, не official engineering determination и не утверждение, что место dangerous.

## Approved wording

Use this wording:

```text
This location has infrastructure risk indicators and should be reviewed on-site.
```

Do not use wording that asserts a location is unsafe or definitively hazardous, or that frames the output as crash prediction. (The exact banned phrasings are intentionally not reproduced here so they never appear un-negated in extracted text.)

## Главная идея

GeoReview Studio — это dashboard/workbench, который показывает:

- review queue;
- map;
- selected candidate detail;
- compact workflow sidebar;
- evidence and data-quality information;
- portfolio-ready artifacts.

Advanced evidence/publication operations should stay behind explicit toggles. The first screen should feel like a real workbench, not a marketing page.

- Source GIS data is read-only.
- Do not delete, move, rename, or overwrite original GIS files.
- Missing OSM tags are data-quality flags, not proof of absent infrastructure.
- Keep infrastructure risk indicators separate from unknown/missing data.
