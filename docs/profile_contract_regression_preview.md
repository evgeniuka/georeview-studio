# Profile Contract Regression Preview

v041 adds a preview-only regression gate for proposed profile mapper config changes.

## Purpose

The regression preview reads the proposed mapper config preview generated by the guarded apply proposal and checks:

- proposed config JSON is parseable;
- proposed hash matches the apply proposal;
- required contract fields are preserved;
- every contract has required fields;
- profile IDs are not duplicated;
- existing profiles are preserved;
- promoted profile is present;
- approved review wording and missing-data policy remain present.

## API

- `GET /api/profile-promotion/regression-candidates`
- `POST /api/profile-promotion/regression-preview`
- `GET /api/profile-promotion/regression-previews`
- `GET /api/profile-promotion/regression-previews/{preview_id}`

## Safety Boundary

This workflow does not modify `profile_mapper_contracts_v001.json`.

It writes only JSON and Markdown preview artifacts under `analysis_output/georeview_studio_profile_promotions`.

Source GIS files remain read-only.
